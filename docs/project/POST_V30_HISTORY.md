## Methodology Shift From Spike-Based To Sprint-Based

TASK-SPECTIX-001 exposed a product lifecycle gap: broad extraction could process all documents while pass 1 remained `in_progress`. That changed the planning model.

D-019, recorded in `docs/DECISIONS.md` by PR #51, split extraction work into a contract-first and route-implementation sequence:

- SPRINT-001: pass lifecycle completion after document processing.
- SPRINT-002A: normalized extraction contracts and runtime guards.
- SPRINT-002B: dedicated priority subtype extraction routes.
- SPRINT-003A: synthesis data model.

Spikes #00 through #03ד remain canonical historical identifiers. New product work uses the SPRINT-N format.
